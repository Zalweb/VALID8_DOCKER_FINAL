What the bug is:
- Browser error: "No 'Access-Control-Allow-Origin' header is present" and frontend shows "Failed to fetch" when calling http://localhost:8000/users/

Root cause (most likely):
- The browser blocked the request because the response had no CORS header. Possible immediate causes:
  1) Backend not reachable (server down or not bound to localhost) so request fails before FastAPI can add headers.
  2) Request is handled by a different service or a proxy that does not include the FastAPI CORS middleware headers.
  3) Using credentials (cookies) with wildcard origin ("*") can lead to CORS header being rejected by browser when allow_credentials=True.

- `Backend/app/main.py` includes CORSMiddleware with allow_origins=["*"], allow_credentials=True.
- `docker-compose.yml` runs backend on port 8000 and frontend on 5173. Note: compose uses build paths `./backend` and `./frontend` while folders are `Backend` and `Frontend` on disk.

Notes about credentials and tokens:
- If your frontend sends an `Authorization: Bearer <token>` header, this is not considered a browser credential and does not force `allow_credentials` rules — CORS still must allow the origin.
- If the frontend uses `credentials: 'include'` (cookies), `allow_credentials=True` must be set and `allow_origins` cannot be `"*"`.
